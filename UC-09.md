## UC-09 · Assign / remove tags

**Actor:** User tags or untags the current selection.
**Goal:** All selected files (or cursor file if no explicit selection) get the tag applied or removed.

---

### `:tag <name>[@<type>]`

1. User types `tag <name>[@<type>]` in command mode and presses Enter.
   - `<name>` is the tag name (autocompleted from existing tags; or new).
   - `@<type>` is optional:
     - If omitted and the tag already exists → reuse it with its existing type.
     - If omitted and the tag is new → create it as type `event`.
   - Typing `@` in the argument switches autocomplete to tag types.
   - Up / Down cycle through suggestions; Tab fills in the highlighted suggestion.
2. If `@<type>` is given and the tag already exists with a **different** type, an error is shown and nothing changes.
3. Already-tagged files are silently skipped (INSERT OR IGNORE).
4. File list reloads; status bar shows `tagged N file(s) with name@type`.

---

### `:untag [name …]`

1. User types `untag` optionally followed by one or more space-separated tag names.
   - No names → remove **all** tags from the targeted files.
   - One or more names → remove only those tags (others are preserved).
   - Tab autocompletes the last (incomplete) word from existing tag names and appends a trailing space so the next name can be typed immediately.
2. File list reloads; status bar shows `cleared all tags from N file(s)` or `removed name, … from N file(s)`.
