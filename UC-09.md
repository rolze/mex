## UC-09 · Assign tag

**Actor:** User tags the current selection with a tag name and optional type.
**Goal:** All selected files (or cursor file if no explicit selection) get the new tag.

**Main Flow:**
1. User enters command mode (`:`) and types `tag <name>[@<type>]`, then presses Enter.
   - `<name>` is the tag name (autocompleted from existing tags; or new).
   - `@<type>` is optional:
     - If omitted and the tag already exists → reuse it with its existing type.
     - If omitted and the tag is new → create it as type `event`.
   - Typing `@` in the argument switches autocomplete to tag types.
   - Up / Down cycle through suggestions; Tab fills in the highlighted suggestion.
2. If `@<type>` is given and the tag already exists with a **different** type, an error is shown and nothing changes.
3. All selected files are attached to the tag (INSERT OR IGNORE — already-tagged files are silently skipped).
4. File list reloads from DB; a status message reports `tagged N file(s) with name@type`.
