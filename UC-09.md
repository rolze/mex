## UC-09 · Assign tag

**Actor:** User tags the current selection with a tag name and optional type.
**Goal:** All selected files (or cursor file if no explicit selection) get the new tag.

**Main Flow:**
1. User enters command mode (`:`) and types `tag <name>[@<type>]`, then presses Enter.
   - `<name>` is the tag name (autocompleted from existing tags; or new).
   - `@<type>` is optional — defaults to `event` when omitted.
   - Typing `@` in the argument switches autocomplete to tag types.
   - Up / Down cycle through suggestions; Tab fills in the highlighted suggestion.
2. If the tag name does not exist it is created with the given type.
3. If the tag name already exists with the same type, it is reused.
4. If the tag name already exists with a **different** type, an error is shown and nothing changes.
5. All selected files are attached to the tag (INSERT OR IGNORE — already-tagged files are silently skipped).
6. File list reloads from DB; a status message reports success or the error.
