## UC-08 В· Smart import of new files

**Actor:** User connects a new drive / phone and integrates all new media files  
**Goal:** new media files are slug-normalised, dated, deduplicated, and copied to the target tree; each import session is tagged

---

### Command

```
:import <path>
```

`<path>` must be an existing directory. The target root is taken from `config.target_dir`.

---

### Flow

1. **Scan** вҖ” runs in a background thread:
   - Walks `<path>` recursively; skips quality-variant dirs (`small`, `web`, `thumbnail`, вҖҰ), junk folders (DCIM, `\d{3}CANON`, MISC), and always-skip file types (`.txt`, `.db`, `.ini`, вҖҰ).
   - For each media file: compute SHA-256 вҶ’ check against existing DB hashes (status=`moved`) вҶ’ derive date вҶ’ derive slug.
   - Date derivation priority: EXIF DateTimeOriginal / DateTime / DateTimeDigitized вҶ’ XMP sidecar вҶ’ filename patterns (yyyyMMdd, yyyy-MM-dd, DD-MM-YY_HHMM mobile) вҶ’ folder path date.
   - Slug derivation: ancestor-folder walk up to 4 tokens, transliterating umlauts (ГӨвҶ’ae, Г¶вҶ’oe, ГјвҶ’ue, ГҹвҶ’ss), stripping junk words and camera codes.
   - Sends `ImportMsg::ScanDone(Vec<ImportEntry>)` on completion or `ImportMsg::ScanError(msg)` on failure.
   - Simultaneously applies folder-mtime consensus (normalise year-month within each source folder using EXIF majority).

2. **Preview screen** вҖ” full-screen TUI dry run:
   - Stats bar: total / pending / duplicate / skipped / unknown-date counts.
   - Scrollable list: `source_basename вҶ’ target_path | date_src | slug_src | status` for every entry.
   - Footer: `[y/Enter] confirm  [Esc] cancel`.

3. **Execute** вҖ” on confirmation, runs in a background thread:
   - `assign_counters()` вҖ” for each unique `YYYY/date-slug` prefix: query `MAX(counter)` from DB (status=`moved`) AND scan existing filesystem dir; take max+1 as 4-digit counter.
   - For each `pending` entry: `fs::copy` вҶ’ verify SHA-256 hash matches вҶ’ on mismatch retry once вҶ’ on second failure mark as error.
   - DB `INSERT OR REPLACE INTO media` with all columns, `status='moved'`.
   - After all files: create/find tag `import-YY-MM-DD` (type `mex`), assign to all newly imported media rows.
   - Sends `ImportMsg::CopyProgress { done, total }` updates and `ImportMsg::CopyDone(ImportSummary)` on finish.

4. **Done** вҖ” media list reloads; status bar shows summary (`N copied, M duplicates, K skipped`).

---

### Data types (src/import.rs)

| Type | Values |
|------|--------|
| `ImportStatus` | `Pending`, `Duplicate`, `Skipped`, `UnknownDate` |
| `DateSource` | `Exif`, `Xmp`, `Filename`, `Folder`, `OsDate`, `Unknown` |
| `DatePrecision` | `Full`, `YearMonth`, `YearOnly` |
| `ImportMsg` | `ScanDone`, `ScanError`, `CopyProgress`, `CopyDone`, `CopyError` |
| `ImportState` (app) | `Idle`, `Scanning`, `Preview(Vec<ImportEntry>)`, `Copying{done,total}`, `Done(String)` |

---

### DB interaction

- Reads `content_hash` (status=`moved`) at scan time for duplicate detection.
- `INSERT OR REPLACE INTO media` вҖ” all existing columns, no schema changes needed.
- Import tag: `import-YY-MM-DD` with type `mex` (2-digit year, e.g. `import-25-01-15`).
