## UC-03 · File Details

**Actor:** User  
**Goal:** See context and metadata of the selected media file at a glance.

**Filename convention** (from MEX.md):
```
<yyyy>/  yyyy-MM-<slug>-####-<caption>.<ext>   ← slug + caption
         yyyy-MM-<slug>-####.<ext>              ← slug only
         yyyy-MM-DD-<caption>.<ext>             ← caption only
         yyyy-MM-DD-####.<ext>                  ← counter only
```

**Main Flow:**
1. User selects a file in the list.
2. List title updates to: `mex — pos / filtered / total`
   - no active filter: `pos / total`
   - active filter: `pos / filtered / total`
3. In the filename column the slug (or caption if no slug) is highlighted cyan+bold so the semantic part stands out from the date prefix and counter.
4. User presses `Enter`/`Space` to open the right-pane preview, which shows:
   - **File** — full relative path
   - **Date** — derived date (yellow)
   - **Ext** — file extension
   - **Tags** — comma-separated (green); `—` if none
   - **Slug** — derived slug (cyan), only shown when present
   - **Capt** — caption slug (cyan), only shown when present

**Acceptance Criteria:**
- List title always shows current position and total; filtered count added when a filter is active.
- Slug/caption highlighted in the filename column; no highlight when both are empty.
- Detail pane shows all fields; slug/caption rows hidden when empty.
