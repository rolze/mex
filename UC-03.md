## UC-03 · File Details

**Actor:** User  
**Goal:** See context and metadata of the selected media file at a glance.

**Filename convention** (from MEX.md):
```
<yyyy>/  yyyy-MM-<slug>-####-<caption>.<ext>   ← slug + caption
         yyyy-MM-<slug>-####.<ext>              ← slug only
         yyyy-MM-DD-<caption>.<ext>             ← caption only
         yyyy-MM-DD-####.<ext>                  ← counter only
```

**Main Flow:**
1. User selects a file in the list.
2. List title updates to: `mex — pos / total` (no filter) or `pos / filtered / total` (filter active).
3. In the filename column the slug (or caption if no slug) is highlighted cyan+bold so the semantic part stands out from the date prefix and counter.
4. User presses `Enter`/`Space` to open the right-pane preview.
5. The metadata header shows a **two-column layout** (3 lines tall):
   - **Left**: File (full relative path), Date (yellow)
   - **Right**: Tags (green; `—` if none), Slug (cyan; `—` if absent), Caption (cyan; `—` if absent)
   - Ext is omitted — already visible in the filename.
6. Below the metadata header the image is rendered (remaining pane height).

**Acceptance Criteria:**
- List title always shows current position and total; filtered count added when a filter is active.
- Slug/caption highlighted cyan+bold in the filename column; no highlight when both are empty.
- Metadata header is 3 lines, two columns; Ext not shown.
- All fields show `—` when absent (never blank).
