## UC-00 · Key concepts and UX guidelines

This is the general guideline for to be considered for all usecases.
Keep this file as brief and concise as possible.

## Key ideas

* terminal, keyboard based
* keep it fast and simple
* keep it intuitive for terminal experienced users (vi, tmux, shell, etc.)
* UX: user should get always enough context to figure out what interactions are available and what is expected next by the user
  * heavy use of auto-suggestions
  * dimmed inline help of params, commands, etc.
  * ...
* general layout
  * main component: list of files, details/preview on demand
  * input field (filter, command), bottom left (2/3 of the space)
  * status/feedback box, bottom right (1/3 of available space), 
* modes:
** normal mode
** filter mode
* interactions type
** filter
** commands
** actions
