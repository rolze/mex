## UC-15 · Open image in sem

**Actor:** User  
**Goal:** Open the cursor image in `sem` — a standalone GTK4 image viewer — directly from mex.

---

### Flow

1. User positions the cursor on an image file.
2. User presses `p`.
3. mex checks the file extension. If it is an image (jpg, jpeg, png, gif, webp, bmp), mex spawns `sem` detached:
   ```
   sem <absolute-path> [--tags "tag1,tag2"]
   ```
   The viewer opens in a new window; mex remains fully interactive.
4. sem displays the image scaled to fit the window, with the filename and tags shown below.
5. On error (file missing, `sem` not in PATH) a status message is shown in the mex status bar.

### `p` key dispatch (type-aware)

| File type | Action |
|-----------|--------|
| Image (jpg, jpeg, png, gif, webp, bmp) | Open in `sem` (this UC) |
| Video (mp4, mkv, …) | Open in mpv (UC-13) |
| Other | Status error: "view: unsupported file type" |

### sem CLI

```
sem <path> [--tags <comma-separated-tags>]
```

- `path` — absolute path to the image; sem exits with an error message if the file is missing.
- `--tags` — optional comma-separated tag list passed by mex; displayed below the image.

### sem window

- GTK4 + libadwaita `ApplicationWindow`, 800×600 default, resizable.
- Header bar shows the filename as the window title.
- Image fills the available area (`ContentFit::Contain`).
- Caption below image: `<filename>  ·  <tag1>, <tag2>` (tags omitted when empty).
- Close: standard window close button or `Escape`.

### Notes

- The mex TUI is not suspended; mex remains fully interactive while sem is open.
- `s` / `j` / `k` keys in mex still control mpv — they are unaffected.
- Tags are passed from mex at launch time; sem does not read `.mex.db` directly.
