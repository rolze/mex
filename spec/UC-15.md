## UC-15 · Open image in sem

**Actor:** User  
**Goal:** View selected image(s) in `sem` — a standalone GTK4 image viewer — directly from mex.

---

### Flow

#### Single image

1. Cursor is on an image file (no selection, or selection of exactly one image).
2. User presses `p`.
3. mex spawns `sem` detached:
   ```
   sem <absolute-path> [--tags "tag1,tag2"]
   ```
4. sem opens a window showing the image scaled to fit, with filename and tags below.

#### Grid view (≥ 2 images selected)

1. User selects 2+ image files (Space to toggle selection).
2. User presses `p`.
3. mex writes a manifest file to `/tmp/mex-sem-<micros>.txt`:
   ```
   /abs/path/a.jpg\ttag1,tag2
   /abs/path/b.png\ttravel
   ```
4. mex spawns sem detached:
   ```
   sem --files <manifest> --cache-dir <db_path>.cache/
   ```
5. sem opens a grid window showing 256 px thumbnails, loaded progressively.
6. Clicking a thumbnail → transitions to full-size single-image view in the same window.
7. Escape (or back button) in single view → returns to the grid.
8. Escape in grid view → closes the window.

### `p` key dispatch

| Selection | File type | Action |
|-----------|-----------|--------|
| ≥ 2 images | image | Open grid in sem (this UC) |
| 0 or 1 | image | Open single image in sem (this UC) |
| any | video | Open in mpv (UC-13) |
| any | other | Status error: "view: unsupported file type" |

### sem CLI

```
sem <path> [--tags <comma-separated-tags>]
sem --files <manifest> --cache-dir <dir>
```

**Single-image mode:**
- `path` — absolute path; sem exits with an error if missing.
- `--tags` — optional comma-separated tags; displayed below the image.

**Grid mode:**
- `--files` — path to a tab-separated manifest (path `\t` tags, one per line).
- `--cache-dir` — directory for 256 px WebP thumbnail cache.

### Thumbnail cache

- Cache dir: `<db_path>.cache/` (e.g. `/home/user/photos/.mex.db.cache/`).
- Cache key: `SHA-256(canonical_path + NUL + size_le8 + mtime_le8)`, first 16 hex chars + `.webp`.
- Generator: libvips — resize to 256 px longest edge, WebP lossy quality 85.
- Cache hit: file exists → served immediately; cache miss → generated in background thread.

### sem window — single-image mode

- GTK4 + libadwaita `ApplicationWindow`, 800×600 default, resizable.
- Header bar shows the filename as the window title.
- Image fills the available area (`ContentFit::Contain`).
- Caption below image: `<filename>  ·  <tag1>, <tag2>` (tags omitted when empty).
- Close: standard window close button or `Escape`.

### sem window — grid mode

- FlowBox of thumbnail cells inside a ScrolledWindow.
- Each cell: 256×256 thumbnail + filename label (ellipsized).
- Thumbnails loaded progressively; grey placeholder shown until ready.
- Header: title = `"N images"`, back button hidden.
- Click thumbnail → single-image view; header shows filename, back button visible.
- Back button / Escape → returns to grid.

### Notes

- The mex TUI is not suspended; mex remains fully interactive while sem is open.
- `s` / `j` / `k` keys in mex still control mpv — they are unaffected.
- Tags are passed from mex at launch time; sem does not read `.mex.db` directly.
- Manifest files in `/tmp` are not deleted by mex; the OS cleans them eventually.
