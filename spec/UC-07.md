## UC-07 Â· Fix date

**Actor:** User  
**Goal:** Fix the creation/modification date of one or more selected files.

**Main Flow:**
1. User filters and/or selects files (multi-selection, or cursor file if nothing selected).
2. User types `:fix-date <yyyy-mm-dd>` in command mode.
   - Up/Down cycles through command-name suggestions; Tab completes the command name.
   - While typing the command name (before the first space) a dim suffix shows the matching completion â€” identical to tag autocompletion in the filter bar.
3. User presses Enter to execute.
4. For each affected file:
   - The filename's date prefix is replaced with the new date:
     - Day format (`yyyy-MM-DD-rest`): full date replaced.
     - Slug format (`yyyy-MM-slug-rest`): only `yyyy-MM` replaced (day not present in filename).
   - The file is renamed on disk (year folder created if needed).
   - The OS mtime is set to the new date **with the original hh:mm:ss preserved** (read from the file's current mtime; falls back to the stored `os_date` time component; defaults to 00:00:00 if neither is available).
   - DB fields `os_date` (stored as `yyyy-mm-dd HH:MM:SS` with preserved time) and `derived_date` (date only: `yyyy-mm-dd`) and `target_path` are updated.
5. The file list reloads from the DB and the filter is re-applied.
6. A status message (yellow, in the filter bar) confirms success or reports errors.
