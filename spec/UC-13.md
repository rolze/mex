## UC-13 · Integrate with remote-controlled video player mpv

**Actor:** User  
**Goal:** Control video playback in mpv from within the mex TUI without leaving it.

---

### Flow

1. User navigates the file list to a video file.
2. User presses `p` — mex validates the file extension (error if not a video), then opens
   it in mpv. If mpv is not running, mex spawns it automatically and waits for it to be ready.
3. User presses `s` to pause/resume, `j`/`k` to jump to the next/previous video.
   These three keys require mpv to already be running; if not, a status message is shown.
4. On any error (file missing, `mpv` not found) a status message is shown.

### Keyboard controls

Both media keys (where the terminal delivers them) and letter shortcuts work:

| Media key | Letter | Action | Requires mpv running |
|-----------|--------|--------|----------------------|
| ⏯ Play / Pause | — | Toggle play/pause | Yes |
| ⏹ Stop | — | Stop playback (mpv stays alive) | Yes |
| ⏭ Next Track | `j` | Move cursor to next video and open it | Yes |
| ⏮ Prev Track | `k` | Move cursor to previous video and open it | Yes |
| — | `p` | Open cursor file in mpv | No (spawns mpv if needed) |
| — | `s` | Toggle pause/resume | Yes |

Letter shortcuts are active in **normal mode only** (inactive while typing in the filter or command bar).

### Notes

- Only known video extensions are accepted (mp4, mkv, avi, mov, webm, flv, wmv, m4v, ts, m2ts, mpeg, mpg, ogv, 3gp, divx, rmvb); other files show an error.
- Multi-selection is ignored; always opens the cursor file.
- The TUI is not suspended; mex remains fully interactive while mpv is open.
- Next/Prev skip non-video files and wrap around the filtered list.
