## UC-06 · Create view of selection

**Actor:** User  
**Goal:** Materialise the current selection (or filtered list) as a named, flat directory of hard links — an on-demand album that can be recreated instantly at any time.

**Preconditions:**

* Media files reside on a Linux-native filesystem that supports hard linking.
* `views_root` is configured in `~/.config/mex/config.toml`; this is the parent directory where view directories are created.

**Main Flow:**

1. User filters and/or selects files.
2. User runs `:create-view <name>`.
3. Target files are determined: the explicit selection if non-empty, otherwise the entire filtered list.
4. The view directory `<views_root>/<name>/` is (re-)created fresh — if it already exists it is deleted first.
5. Each target file is hard-linked as `<views_root>/<name>/<basename>` (flat layout, no subdirectories).
6. A status message reports the number of files linked and the full view path.

**Error cases (reported via status message):**

* `views_root` not configured.
* Name argument missing.
* Filtered list is empty (nothing to link).
* Filesystem errors during hard link (individual failures logged to stderr; summary shown).

**Notes:**

* File basenames are globally unique (year-prefixed convention), so name collisions cannot occur.
* Hard linking is instant regardless of file size.
* A view can be recreated at any time by running the command again with the same name.
