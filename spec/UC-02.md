## UC-02 · Browse Media Files

**Actor:** User  
**Goal:** Navigate the indexed media collection in the TUI with keyboard controls.

**Preconditions:** `.mex.db` exists and `target_root` is configured (UC-01).

**Main Flow:**
1. Split-pane TUI: left = file list + filter bar; right = image preview (UC-03).
2. File list shows fixed-width columns: **folder** (6 chars), **filename** (remaining space, front-truncated with `…`), **tags** (30 chars, end-truncated).
3. Slug and caption are highlighted in the filename column per UC-03 rules.
4. User navigates with:
   - `↑` / `↓` — move cursor one row
   - `PgUp` / `PgDn` — full page
   - `Ctrl-u` / `Ctrl-d` — half page
   - `Home` / `End` — jump group-by-group (UC-04)
   - `Enter` — toggle right-pane preview
   - `Space` — toggle file selection (UC-04)
   - `Ctrl-A` — select all / deselect all in the current result set
   - `Esc` — cancel command → clear selection → close preview → clear filter
   - `:` — enter command mode (e.g. `:q` to quit)
5. All printable keys (except `:`) feed the live text filter; `#` prefix enters tag mode (UC-05). A `*` character in the text filter acts as a wildcard matching **one or more** characters; e.g. `2005*mp4` matches any filename that contains `2005` followed by at least one character and then `mp4`. Without `*` the filter is a plain substring match.
6. The filter bar (bottom) shows active filter state: `/text` (white bold) + `#tag` tokens (cyan bold).
7. When a text filter is active, matching characters in the **filename** column are highlighted with a distinct background colour. For wildcard patterns the matched segments (not the wildcard gaps) are highlighted.
8. List title: `mex — pos / total` (no filter) or `pos / filtered / total` (filter active); appends `(N selected)` when applicable.
9. Files whose physical path is absent under `target_root` are shown with a `!` marker (in place of the selection dot) and a reddish row tint (`Rgb(60,15,15)` background, `Rgb(220,100,100)` foreground). The `missing_on_disk` flag is persisted in the DB and survives across sessions. It is checked — and cleared if the file reappears — lazily each time the terminal preview (right pane) is opened for that file.

**Acceptance Criteria:**
- Navigation is instant (no visible lag on lists up to 50 000 entries).
- Cursor row is always kept visible (auto-scroll).
- Columns have fixed widths; filenames exceeding their column are truncated at the front (tail preserved).
- Matched filter text is highlighted in the filename column with a different background; no highlight when filter is empty. For wildcard patterns each matched segment is highlighted individually.
- Wildcard filtering with `*` is instant (no visible lag) on lists up to 50 000 entries.
- Files absent from `target_root` display a `!` marker and reddish tint; trashed files retain their existing dim/gray styling and take priority over missing styling.
