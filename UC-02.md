## UC-02 · Browse Media Files

**Actor:** User  
**Goal:** Navigate the indexed media collection in the TUI with keyboard controls.

**Preconditions:** 
* we have an existing sqlite DB to be used, with tags and album views based on tags
  * media DB: .mex.db
  * folder with assets: <extract a small sample ~50 files and folders from DB, target path, prefix it with mex-media-root>

**Main Flow:**
1. The main left view displays the existing folder structure, media assets sorted by name
2. User navigates with:
   - `↑` / `↓` — move selection up / down
   - `PgUp` / `PgDn` — skip a full page up / down
   - `Ctrl-u` / `Ctrl-d` — skip half a page up / down
   - `Home` / `End` — jump to first / last item
   - `Enter` / `Space` — toggle right-pane preview
   - `Esc` — close preview / clear filter / cancel command
   - `:` — enter command mode (e.g. `:q` to quit)
3. All letter keys feed the live search filter immediately — no mode switching needed.
4. The list shows fixed-width columns: **folder**, **filename** (front-truncated with `…` if too long), **tags**

**Acceptance Criteria:**
- Navigation feels instant (no visible lag on lists up to 50 000 entries).
- Selected item is always visible (auto-scroll) on the right pane
- Columns have fixed widths; filenames that exceed their column are truncated at the front (tail preserved)
- Tags column is shown instead of date

**Design choices:**
* split pane
  * left: search field at the top, quick filtering in fzf manner (using "fzf" ?), show existing folder layout
  * right: quick preview of image
* Column layout (left pane, fixed widths):
  * folder: 6 chars  
  * filename: remaining space (front-truncated)
  * tags: 30 chars (end-truncated)
